# 人物

<!-- 本文为功能参考文档，面向普通用户。截图占位符为 `TODO: 截图`，后续补充。 -->

人物功能把人脸识别结果整理成可浏览的人物集合。你可以为识别出的人物命名、隐藏不需要展示的人物、纠正错误的人脸归属，并按人物浏览或分享相关照片。

::: tip 提示
人物识别依赖 AI / ML 功能。管理员可在[设置 → AI](/zh-cn/user-manual/features/settings#ai) 中启用或关闭人物识别。
:::

## 人物列表

人物入口位于[合集](/zh-cn/user-manual/features/collections)页面。人物列表以头像卡片展示识别出的人物，并显示对应照片数量。

页面顶部包含：

- **仓库范围** — 用于限定列表中展示哪些仓库里的照片结果。
- **可见 / 全部** — 默认只显示未隐藏的人物；切换到“全部”后也会显示已隐藏的人物。

单击人物卡片会进入人物详情页。

## 人物详情

人物详情页顶部展示人物头像、名称、照片数量、成员数量和更新时间。下方是该人物相关照片的资源浏览视图，使用与[资源库](/zh-cn/user-manual/features/assets)相同的浏览组件。

可执行的常用操作：

- **编辑** — 打开人物编辑弹窗，修改名称、隐藏状态、脸部归属或合并人物。
- **分享** — 为该人物对应的照片集合创建公开分享链接。
- **选择照片后分享** — 只分享当前选中的照片，而不是整个人物集合。

人物详情本身不按仓库限制。即使人物的照片跨多个仓库，也会作为同一个人物展示。

## 命名与隐藏

在人物详情页单击**编辑**后，默认进入“信息”标签页：

- **名称** — 为人物设置易识别的名字。保存后，该人物会被视为已确认。
- **隐藏** — 将人物从默认人物列表中隐藏。隐藏不会删除照片，也不会改变人脸归属；在人物列表切换到“全部”后仍可看到。

隐藏适合处理误识别、路人、重复但暂时不想整理的人物。

## 纠正人脸

编辑弹窗中的“脸部”标签页展示该人物下的所有人脸裁剪图。这里处理的是“哪一张脸属于谁”，不会修改原始照片。

你可以：

- **设置封面** — 在单张脸的菜单中将其设为人物头像。
- **选择多张脸** — 进入选择模式后勾选多个脸部裁剪图。
- **移动到另一个人物** — 将选中的脸归到另一个人物。
- **从此人物移除** — 将选中的脸从当前人物中移除，等待后续重新整理。

::: warning 注意
移除人脸只会解除它与当前人物的关联，不会删除照片或裁剪源图。
:::

## 合并人物

如果同一个人被识别成多个不同人物，可以在编辑弹窗的“合并”标签页中处理。

操作方式：

1. 打开要保留的人物详情页。
2. 单击**编辑**，切换到“合并”。
3. 搜索并选择要并入当前人物的其他人物。
4. 单击“合并”。

合并后，来源人物会并入当前人物。照片仍保留在资源库中，人物筛选和人物列表会随之更新。

## 分享人物照片

人物详情页顶部的**分享**按钮会为整个人物照片集合创建公开分享链接。服务端会根据人物 ID 解析分享范围，不需要你先手动全选。

如果只想分享其中几张照片，可以在下方资源浏览区进入选择模式，选中照片后使用批量操作里的**分享**。分享链接的创建、下载权限和管理方式见[分享](/zh-cn/user-manual/features/sharing)。

## 重建人物识别

当新增照片后，管理员可以通过相关维护入口触发人物识别重建。重建会重新运行人脸聚类，用于更新人物列表和照片归属。

已做过的手动纠正会尽量保留：例如命名、移动人脸、合并人物等操作不会因为普通重建而简单丢失。
