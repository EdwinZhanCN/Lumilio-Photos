# Lumen AI

::: warning 注意
流明集当前处于 Beta 阶段。请先使用测试媒体或已有可靠备份的资料库进行试用，不要将本应用作为重要媒体的唯一存储位置。
:::

::: danger 仅限受信网络
当前 Lumen 节点连接不提供传输加密或节点身份认证。不要将 Lumen Hub、Host Broker 或推理服务端口直接暴露到互联网或不可信网络。自动发现只能说明节点可见，不能代替网络访问控制或节点信任判断。
:::

Lumen 与基础媒体管理相互独立。未启用或节点暂时不可达时，上传、扫描、浏览、相册和手动整理仍可使用；语义、人物、OCR 和物种识别等依赖模型的任务会等待节点恢复。

| 连接方式 | 推理输入发送到哪里 | 数据形式 | 传输边界 |
| --- | --- | --- | --- |
| Desktop 本机 Hub | 本机回环地址 | tensor 或处理后的图像 | 不经过局域网 |
| mDNS / 静态节点 | 所选自托管 Hub | 语义与 BioCLIP 可用 tensor；回退、OCR 与人物使用处理后的图像 | 当前无传输加密或节点认证，仅限受信网络 |
| Host Broker | Broker 只提供节点列表；推理输入仍直达所选 Hub | 与目标 Hub 的任务路径相同 | Broker 不代理或保护推理流量 |

这里的“处理后的图像”仍可能包含可识别的画面内容；“tensor”也不应被当作匿名化或加密数据。

## Desktop：启用本机 Hub

1. 在 Desktop 控制面板中启用本地 Lumen，并选择适合设备的模型预设与计算后端。
2. 等待 Hub 组件和模型下载完成。Desktop 会生成配置、启动进程并监督运行状态。
3. 打开流明集的“服务监控”，确认本机节点已连接，并检查它公开的能力是否包含你要使用的任务。
4. 在“设置 → AI”启用对应 ML 功能，再用少量测试照片验证结果。

受 Desktop 管理的 Hub 绑定本机回环地址，推理输入不会发送到局域网节点。不同预设包含的模型不同；若服务监控中缺少某项能力，请先核对预设，不要把任务未运行直接判断为故障。

## Docker 与其他自托管节点

Docker 部署可在安装页的 Compose 配置器中选择：

- **暂不启用**：只运行基础媒体管理。
- **Linux Host network + mDNS**：容器直接参与宿主机网络发现，适合受信任的 Linux 局域网。
- **Lumen Host Broker**：由宿主机发现节点，再把节点列表提供给容器；Broker 不代理媒体或推理请求。
- **静态节点地址**：直接填写一个或多个可达的 `host:port`。

macOS 与 Windows 的 Docker Desktop 通常无法让容器直接参与宿主机局域网 mDNS，优先评估 Host Broker 或静态地址。静态地址可以指向任意网络可达的节点，但当前连接安全边界不适合穿越互联网或不可信网络。

## 验证与故障排查

按以下顺序检查：

1. 在服务监控中确认节点在线，并查看任务能力，而不只看节点是否被发现。
2. 确认流明集所在主机可以直接访问 Hub 的推理端口；Host Broker 可见不代表推理端口可达。
3. 检查防火墙、客户端隔离、VLAN 和容器网络。mDNS 通常不会跨子网。
4. 确认“设置 → AI”已启用所需 ML 功能，并等待现有后台队列完成。
5. 使用少量测试照片重试。AI 输出需要复核，不应据此自动删除或覆盖原件。

语义图像分析与 BioCLIP 在能力匹配时会使用更紧凑的 tensor 路径；协议不匹配或预处理失败时会自动回退到兼容图像路径。OCR 与人物任务当前使用图像路径。这是**协议兼容回退**，只用于保持任务可用，不提供传输安全，也不会把不受信任的远程节点变成本地或加密节点。

更多模型与数据流边界见[AI 与 Lumen](../introduction/lumen.md)。
