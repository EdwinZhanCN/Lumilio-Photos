# Lumilio Agent

Lumilio Agent 是内置的 AI 助手，用来围绕你的照片库进行对话、整理、分析和策展。它可以结合当前页面上下文、你手动提到的人物或相册，以及 Agent 返回的结果看板，帮助你继续探索照片集合。

::: tip 提示
Lumilio Agent 需要管理员在[设置 → AI](/zh-cn/user-manual/features/settings#ai) 中启用并配置对应模型服务。未启用或未配置时，聊天面板会显示提示并提供前往 AI 设置的入口。
:::

## 入口

你可以从两个位置使用 Lumilio Agent：

- **独立页面** — 打开 `/lumilio`，进入完整的 Agent 看板页面。
- **资源查看场景** — 在资源库或全屏查看器中打开聊天面板，围绕当前可见资源继续提问。

独立页面由两部分组成：

- **看板** — 用来保存和排列 Agent 生成的结果卡片。
- **聊天面板** — 用来发送问题、选择模式、引用上下文和查看回答。

## 聊天与模式

聊天面板支持普通自然语言输入，也支持通过快捷模式限定 Agent 的工作方向。

当前模式包括：

| 模式 | 适合场景 |
|------|----------|
| **回顾（Review）** | 回顾某段时间的拍摄历程 |
| **整理（Organize）** | 整理照片、分组、打标签或创建相册 |
| **分析（Analyze）** | 分析拍摄习惯、器材和趋势 |
| **精选（Curate）** | 从一组照片中挑选更值得保留或展示的作品 |

可以在空白聊天页点击模式卡片，也可以在输入框中输入 `/` 打开模式菜单。选择模式后，它会保持生效，直到你手动清除。

## 引用上下文

输入框支持 `@` 提及，帮助 Agent 明确你要讨论的对象。当前可提及：

- **人物**
- **相册**
- **看板中的 pin**
- **相机型号**
- **镜头**

输入 `@` 后先选择类型，再搜索具体对象。被提及的对象会随消息一起发送，减少歧义。

聊天面板还会显示当前页面贡献的上下文标签，例如当前资源、当前资源集合或当前查看范围。你可以移除不想发送的上下文标签。

## 回答内容

Agent 的回答可能包含：

- 普通文本说明。
- 推理过程摘要。
- 工具调用状态。
- 需要你确认的操作。
- 可视化结果卡片。

如果上一次模型调用返回了 token 用量，聊天标题区域会显示简短的 token 统计。

## 看板与 pin

当 Agent 返回可视化结果卡片时，可以将其 pin 到看板。pin 会把当时的结果快照保存下来，即使之后开启新对话，也能在看板中继续查看。

看板支持：

- **拖动移动** — 调整卡片在看板中的位置。
- **整理** — 将卡片向上压缩排列。
- **切换视图** — 在封面、数字、趋势、马赛克等视图之间切换。
- **调整尺寸** — 通过卡片菜单选择小、中、大尺寸。
- **重命名** — 修改卡片标题。
- **删除** — 从看板移除 pin。
- **打开资源集合** — 单击卡片主体进入对应的资源视图。

::: info 说明
看板卡片是已保存的结果入口，不是普通相册。它用于持续查看某次 Agent 结果，真正的资源管理仍在资源库、相册、标签和工具页面完成。
:::

## 新对话

聊天面板顶部的新对话按钮会清空当前对话显示，便于开始新的任务。已经 pin 到看板的结果不会因为开启新对话而消失。

## 使用建议

- 想讨论当前页面看到的照片时，先确认上下文标签是否正确。
- 想让 Agent 聚焦某个人、相册或器材时，使用 `@` 提及。
- 想让 Agent 按某类任务行动时，先选择 `/` 模式。
- 重要结果可以 pin 到看板，避免在长对话中反复查找。
