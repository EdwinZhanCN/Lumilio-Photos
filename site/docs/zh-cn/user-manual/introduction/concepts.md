# 核心概念

::: warning 注意
流明集当前处于 Beta 阶段。请先使用测试媒体或已有可靠备份的资料库进行试用，不要将本应用作为重要媒体的唯一存储位置。
:::

本页统一后续文档中使用的常见名称。在流明集中，**媒体**是对照片、视频和音频内容的统称；**资源**则是流明集为其建立的管理记录。

本文偶尔使用“资料库”表示用户整体拥有的一组媒体及其应用数据。它不是界面中的“资源库”，也不是磁盘上的单个“仓库”。

## 媒体与存储

### 原件

仓库中保存的源媒体文件。流明集的普通管理和工作室调整默认不覆盖原件。

### 存储根目录

首次设置时选择的统一存储位置，也称为 `storage root` 或 `root`。流明集的仓库位于该目录下。

### 仓库

位于存储根目录下、用于保存和管理媒体的独立子目录。一个实例可以有多个仓库。

### 主仓库

首次设置时强制创建在 `<root>/primary/` 的默认仓库。当操作没有指定其他仓库时，通常使用主仓库。

### 上传

将设备或浏览器中选择的媒体复制到仓库，然后建立资源记录并开始后台处理。

### 扫描

检查仓库目录中已有的媒体，并在不移动原文件的情况下建立或更新资源记录。

### 云端导入

从受支持的云端来源下载媒体，并将下载内容保存到本地仓库。某些来源可能属于实验性功能。

仓库布局和三种媒体进入方式的区别，请参阅[媒体与仓库](./repositories.md)。

## 浏览与组织

### 资源库

浏览、筛选和检查媒体的主要界面。它可以展示一个或多个仓库中的资源。

### 相册

由用户选择媒体组成的组织方式。将资源加入或移出相册，不会移动对应的原件。

### 合集

浏览相册、人物、地点及其他媒体聚合视图的入口。

### 人物

基于可选人脸分析结果形成的照片分组。识别结果可能出错，用户可以对分组进行修正。

### 堆叠

将连拍、相关原件或手动选择的多个资源，作为一个逻辑项目展示。堆叠只改变展示和组织方式。

## 处理与服务

### 后台任务

流明集在后台执行的异步工作，例如提取元数据、生成缩略图、转码、扫描和智能索引。媒体出现在资源库后，某些结果仍可能需要时间才会完成。

### Lumen

为流明集提供可选 AI 与机器学习能力的本地服务，包括基于 SigLIP 2 的图像语义分析、基于 InsightFace 的人物识别、基于 PP-OCRv6 的文字识别，以及基于 BioCLIP 2 与 TreeOfLife200M 的物种识别。未连接 Lumen 时，基础的媒体存储、浏览和组织功能仍可使用。

### 工作室

用于调整媒体并导出新文件的编辑区域。编辑记录与原件分开保存。

### 回收站

用于查看和恢复已软删除资源的区域。当前 Beta 版本尚未提供面向用户的永久删除功能。

### 分享

按用户授权为其他人提供媒体浏览或下载访问。具体能力取决于分享时选择的权限。

### 管理与服务监控

“管理”用于仓库、扫描和处理任务；“服务监控”用于查看队列、AI 节点和系统能力状态。
