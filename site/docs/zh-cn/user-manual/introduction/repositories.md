# 存储位置与仓库

::: warning 注意
流明集当前处于 Beta 阶段。请先使用测试媒体或已有可靠备份的资料库进行试用，不要将本应用作为重要媒体的唯一存储位置。
:::

流明集把**允许保存仓库的目录**与**实际保存媒体的仓库**分成两个概念：

- **存储位置（Storage Location）**是流明集获准使用的父目录，使用 `.lumilioroot` 保存可移动的目录身份。
- **仓库（Repository）**是保存媒体原件和仓库配置的独立目录，使用 `.lumiliorepo` 保存仓库身份。

一个存储位置可以包含多个仓库，但同一个目录不能同时既是存储位置又是仓库。

```text
存储位置/
├─ .lumilioroot
├─ primary/
│  ├─ .lumiliorepo
│  └─ .lumilio/
└─ 家庭照片/
   ├─ .lumiliorepo
   └─ .lumilio/
```

## 三个容易混淆的操作

| 操作 | 在哪里执行 | 选择什么目录 | 是否初始化仓库 |
| --- | --- | --- | --- |
| **添加位置** | Desktop 控制面板 | 一个现有父目录，例如外置硬盘中的 `Lumilio/` | 否；只登记或创建 `.lumilioroot` |
| **创建仓库** | 浏览器中的仓库创建界面 | 从已登记且在线的存储位置列表中选择 | 是；在该位置下创建子目录和 `.lumiliorepo` |
| **挂载资源库** | Desktop 控制面板 | 已经包含 `.lumiliorepo` 的仓库目录 | 否；只登记已有仓库 |

::: tip 要在外置硬盘上创建一个新仓库
先在 Desktop 控制面板选择外置硬盘中的**父目录**并执行“添加位置”，再回到浏览器创建仓库并选择这个存储位置。不要把一个空目录交给“挂载资源库”；该操作只接受已有仓库。
:::

## 添加存储位置不会创建仓库

“添加位置”表示授权流明集使用一个目录作为仓库容器。目录中没有 `.lumilioroot` 时，Desktop 会为它创建一个；已有标记则用于识别重新挂载或路径变化后的同一个位置。

添加完成后，该位置会出现在浏览器的新建仓库表单中。创建仓库时可以选择：

- 存储位置；
- 文件布局：按拍摄日期、单目录或内容寻址；
- 同名文件策略：自动重命名、唯一 ID 或覆盖；
- 可选的云端来源凭据。

本地创建与云端创建使用同一套位置和文件策略；云端创建只额外绑定云端凭据。

主仓库是例外：它在首次设置中创建，并且只能位于不可移除的默认存储位置。普通仓库可以创建在任意已登记且在线的存储位置中。

## 挂载资源库只用于已有仓库

“挂载资源库”用于把磁盘上已有的 Lumilio 仓库登记到当前实例。选择的目录必须包含有效的 `.lumiliorepo`；空目录和普通照片文件夹不会被自动初始化。

如果同一个仓库身份已经登记在另一条路径，Desktop 会要求明确选择：

- **作为移动后的位置**：原仓库换了磁盘、挂载点或 Windows 盘符；
- **作为副本登记**：这是需要独立身份的复制目录。

流明集不会仅根据旧路径是否在线来猜测“移动”还是“复制”。

## 媒体如何进入仓库

| 方式 | 流明集的处理 | 对来源文件的影响 |
| --- | --- | --- |
| 上传 | 先写入仓库内的暂存区，再提交到正式存储位置 | 在仓库中产生一份由流明集管理的文件；不删除上传端来源文件 |
| 扫描 | 读取仓库中已有的支持文件，计算内容指纹并建立资源记录 | 原位登记，不因扫描而移动已有文件 |
| 云端导入 | 下载到仓库暂存区，再提交至正式存储位置 | 在本地仓库中保存下载内容；流明集不会主动删除云端原件 |

上传或导入遇到同名文件时，行为由创建仓库时选择的同名文件策略决定。试用阶段建议保持默认的自动重命名策略。

## 位置离线、移动和移除

外置磁盘或网络卷断开后，存储位置及其仓库会显示为离线。流明集保留其身份和浏览记录，但在重新连接前拒绝上传、导入、设置修改等写入操作，不会在另一块磁盘上静默创建同名目录。

当同一个 `.lumilioroot` 因磁盘挂载点或 Windows 盘符变化出现在新路径时，在 Desktop 控制面板重新添加该目录，并确认“重新连接到此处”。位置内仓库的相对路径会随之更新。

移除一个未被仓库使用的外部存储位置，只解除当前实例中的登记；不会删除目录、标记或媒体。仍有已登记仓库引用的位置不能直接移除。

## 哪些数据不会跟随存储位置

存储位置不是完整工作区。以下内容属于运行流明集的机器，不会写入或跟随 `.lumilioroot` 移动：

- PostgreSQL 数据库；
- 登录密钥和数据库凭据；
- 云端会话与凭据状态；
- 服务日志、Lumen 模型和数据库备份。

仓库自己的可恢复工作目录位于 `.lumilio/`，包括导入暂存和非破坏性编辑记录。不要手动修改 `.lumilioroot`、`.lumiliorepo` 或 `.lumilio/`。

## 使用前注意事项

- 在导入、扫描或后台处理期间，不要从文件管理器移动、重命名或删除仓库内文件。
- 运行流明集的系统账户必须能够读取仓库；上传、导入和内部管理还需要写入权限。
- Docker 只能使用容器已经挂载并在服务器配置中登记的路径；浏览器不能授权新的宿主机目录。
- 在移动仓库、更换磁盘或调整挂载路径前，先停止相关任务并确认媒体与数据库均有独立备份。

::: tip 建议的第一次试用方式
先使用默认存储位置中的主仓库完成[首次使用](./first-use.md)里的上传、删除与恢复验证。需要验证外部存储时，再用少量测试媒体依次尝试“添加位置 → 创建仓库”或“挂载已有资源库”。
:::
