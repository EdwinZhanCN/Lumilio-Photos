# 首次使用

::: warning 注意
流明集当前处于 Beta 阶段。请先使用测试媒体或已有可靠备份的资料库进行试用，不要将本应用作为重要媒体的唯一存储位置。
:::

本页带你使用少量照片完成第一条可验证的使用路径。请先不要导入正式资料库，也不要启用 iCloud 导入。

## 开始前

准备 3–10 张已有可靠备份的测试照片，最好包含不同尺寸、拍摄时间和相机来源。确认默认存储位置所在磁盘具有足够空间，且 Desktop 或 Docker 指向的媒体路径符合你的预期。

## 1. 初始化服务器

第一次打开流明集时，应用会先保护数据库凭据并生成应用密钥。Desktop 将这些内容保存在本机应用数据目录，Docker 将其保存在独立的 app-state 持久化卷；它们不会写入或跟随媒体存储位置移动。该操作只会运行一次。

![流明集服务器初始化页，说明将轮换数据库凭据并生成应用密钥](/images/first-use/server-initialization.png)

选择“开始初始化”，等待页面进入首次运行设置。如果初始化失败，不要反复删除应用数据或 app-state；请先查看日志并检查配置路径与写入权限。

## 2. 开始首次运行设置

向导会依次展示语言与地区、管理员账户、二次验证、Passkey、恢复码和主存储设置。二次验证可暂时跳过；该向导只会在创建首个管理员时出现。

![流明集首次运行向导欢迎页，左侧显示管理员、身份验证器、Passkey、恢复码和存储步骤](/images/first-use/setup-welcome.png)

在欢迎页选择语言和地区，然后开始设置。地区会影响 Desktop 更新与 Lumen 模型的下载来源，之后仍可在 Desktop 控制面板中调整。

## 3. 创建管理员并保护账户

创建第一个管理员账户。请使用唯一且足够长的密码，不要复用其他服务的凭据。

![流明集创建管理员页，包含用户名、密码、密码强度和确认密码字段](/images/first-use/admin-account.png)

强烈建议先配置验证器应用（TOTP），再按需添加 Passkey。当前向导允许“暂时跳过”；跳过 TOTP 时不会进入 Passkey 和恢复码配置，账户将暂时只受密码保护。若页面生成恢复码，请将其保存在密码管理器或其他与流明集分开的安全位置，并在向导中确认已保存。

::: danger 恢复码不应只保存在流明集所在的设备上
如果密码、二次验证设备和恢复码同时丢失，管理员可能无法恢复访问。
:::

## 4. 创建主仓库

默认存储位置已由 Desktop 的本机应用目录或 Docker 媒体挂载配置确定。向导的最后一步会以只读方式显示该路径，并在其 `primary/` 子目录中创建强制的主仓库。请在继续前再次确认；如需更改 Docker 默认位置，应返回部署配置，而不是在向导中填写另一条路径。Desktop 的外部位置应在启动完成后从控制面板添加。

![流明集设置主存储页，显示仓库名称、只读的默认存储位置、日期存储策略和自动重命名策略](/images/first-use/primary-storage.png)

第一次试用建议保持默认设置：

- 存储策略：`date`
- 重复文件处理：`rename`

选择“创建并打开控制台”后，流明集会完成初始设置。存储位置、创建新仓库与挂载已有仓库的区别请参阅[存储位置与仓库](./repositories.md)。

## 5. 上传第一批照片

进入管理页，选择主仓库，然后上传准备好的 3–10 张照片。上传界面显示的文件传输完成，不一定代表后台入库已经完成；请等待相关任务进入完成状态。

随后前往资源库，确认：

- 照片数量与上传数量一致。
- 缩略图可以显示，单张照片可以打开。
- 拍摄时间、尺寸和相机等基础元数据符合预期。
- 原件可以正常读取或下载。

## 6. 验证删除与恢复

选择其中一张测试照片，将其删除，然后前往回收站将其恢复。确认照片重新出现在资源库中，并且原件仍可正常读取。

当前 Beta 版本的普通删除是软删除，但这不能替代备份。请参阅[数据完整性与备份](./integrity.md)。

## 7. 可选：启用 Lumen

在基础媒体路径已经验证后，可以启用 Desktop 管理的本地 Lumen Hub，或连接局域网中的 Lumen 节点。前往服务监控页确认节点和所需能力可用，再为测试照片执行语义分析、人物识别、OCR 文字识别或 BioCLIP 物种识别。

AI 任务会在后台运行，结果可能稍后才出现，且需要用户复核。连接方式请参阅 [AI 与 Lumen](./lumen.md)。

## 扩大资料库前

- 在“管理”和“服务监控”中确认没有持续失败的后台任务。
- 在“设置 → 服务器”中检查数据库自动备份，并至少完成一次立即备份与下载验证；该备份不包含媒体原件，仍需独立备份所有存储位置中的仓库。扩大资料库前，应实际演练适用于当前部署方式的恢复方案。
- 逐步增加媒体数量，不要第一次就导入整个资料库。
- 将 iCloud 等实验性来源作为独立测试，请先阅读[实验性功能与已知风险](./experimental.md)。
- 需要使用外置硬盘时，先在 Desktop 控制面板“添加位置”，再从浏览器创建仓库并选择该位置；“挂载资源库”只用于已经包含 `.lumiliorepo` 的仓库。
