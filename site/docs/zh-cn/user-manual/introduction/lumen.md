# AI 与 Lumen

::: warning 注意
流明集当前处于 Beta 阶段。请先使用测试媒体或已有可靠备份的资料库进行试用，不要将本应用作为重要媒体的唯一存储位置。
:::

Lumen 是流明集的可选自托管 AI 推理服务。Hub 可以运行在安装流明集的计算机上，也可以部署到你明确配置并信任的其他设备。

Lumen 不是基础媒体管理的必要条件。未安装、未连接或节点暂时不可用时，上传、扫描、浏览、相册和其他基础功能仍然可用；只有依赖对应模型的能力会暂时缺席。

## Lumen 能做什么

| 能力 | 模型 | 在流明集中的用途 |
| --- | --- | --- |
| 图像语义分析 | SigLIP 2 | 理解图像语义，为语义检索和相关分析提供向量 |
| 人物能力 | InsightFace | 检测人脸并提取特征，供流明集形成人物分组并由用户修正 |
| OCR 文字识别 | PP-OCRv6 | 识别图像中的文字及其位置 |
| **BioCLIP** 物种识别 | BioCLIP 2 + TreeOfLife200M | 辅助识别照片中的生物类群与物种 |

实际可用能力取决于 Lumen 节点已安装的模型和选择的预设。精简预设只包含语义分析与人物能力，并非每个节点都同时提供表中全部功能。AI 输出是辅助性结果，可能出现误识别、遗漏或不稳定分类；在使用人物、OCR 或物种结果前，请由用户复核。

## Desktop：本地 Hub

流明集 Desktop 可以为用户安装并管理本地 Lumen Hub。首次启用本地 AI 时，Desktop 会下载与当前平台和配置相匹配的 Hub 组件及模型，因此需要稳定的网络、足够的磁盘空间和一定的初始化时间。

启用后，Desktop 会生成本机配置、启动 Hub、等待服务就绪，并在应用运行期间监督该进程。如果用户保持本地 AI 启用，Desktop 会在之后启动时自动恢复它。

受 Desktop 管理的本地 Hub 固定绑定本机回环地址，因此不需要通过 mDNS 发现自身；图片处理和推理都可留在本机。Desktop 仍会启动 mDNS 发现，以便在同一局域网中查找其他 Lumen 节点。

## 局域网自动发现

在普通家庭局域网中，Lumen Hub 可以通过 **mDNS** 广播自己，流明集则自动发现节点并读取其能力。当两者位于同一局域网且网络允许 mDNS 时，通常不需要手动填写节点地址。

使用非本机节点时，流明集会把经过预处理的图像数据或模型输入发送到所选 Hub。当前节点连接不提供传输加密或节点身份认证，因此请只在受信任且受网络访问控制保护的环境中使用 Lumen。不要将 Lumen Hub、Host Broker 或推理服务端口直接暴露到互联网、访客网络或其他不可信网络；自动发现只说明节点可见，不代表节点已经过认证。

mDNS 通常不会跨越不同子网或 VLAN。以下情况也可能导致无法自动发现：

- 设备连接在不同的物理网络、访客 Wi-Fi 或隔离网络中。
- 路由器启用了客户端隔离，或企业网络禁止组播发现。
- 主机防火墙拦截 mDNS 或 Lumen Hub 的服务端口。
- 流明集运行在与宿主机局域网隔离的容器网络中。

如果自动发现失败，请先确认 Lumen Hub 正在运行、两台设备位于同一局域网，并检查防火墙与路由器隔离设置。如果当前网络不适合 mDNS，可以使用 Host Broker 或静态节点地址。

## 性能路径与兼容回退

语义图像分析与 BioCLIP 会在节点能力匹配时由流明集完成预处理，并发送更紧凑的模型输入，以减少 Hub 端重复解码和缩放。若节点不支持对应预处理协议，或本地预处理不可用，流明集会自动回退到兼容的图像路径；两条路径表达同一个任务。OCR 与人物处理当前使用图像路径。

这种能力协商与安全回退让不同模型预设可以共存，也为后续模型升级保留兼容路径。它不代表所有设备都有相同性能；实际速度和资源占用仍取决于模型、CPU/GPU、内存、媒体尺寸与并发任务。

## Docker 环境

Docker 是否能直接使用 mDNS，取决于宿主平台和容器网络模式。

### Linux 宿主机

当流明集容器使用 **Host network** 时，它可以直接参与宿主机的局域网 mDNS 发现。如果使用默认 bridge 网络，mDNS 广播通常不会自动进入容器，应改用 Host Broker 或静态节点。

### macOS 与 Windows Docker Desktop

Docker Desktop 的容器运行在独立的虚拟化网络中，通常不能直接使用宿主机的局域网 mDNS。这种情况下，可在宿主机运行 **Lumen Host Broker**，再让容器中的流明集通过 `host.docker.internal` 连接 Broker。

各平台的适用边界与计划补充的操作内容，请参阅 [Lumen AI](../features/lumen-ai.md)。该功能页目前仍在完善，不应被视为完整的部署或故障排查指南。

::: info 开源组件
**Lumen Hub** 负责加载模型并执行本地推理；**Lumen SDK** 负责节点发现、连接与请求。

**Lumen Host Broker** 在宿主机上发现节点，再将节点变化发布给无法直接使用局域网 mDNS 的应用。它只辅助发现，**不代理媒体内容或推理请求**。
:::
