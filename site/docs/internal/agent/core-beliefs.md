# Core Beliefs

- Local-first media ownership matters more than clever automation.
- Original files should be preserved unless the user explicitly chooses otherwise.
- Repository semantics should be explicit: storage roots, primary repositories, staging, and scans must be easy to reason about.
- AI and ML should enhance organization and search, not become mandatory infrastructure for basic photo management.
- Defaults should boot cleanly in Docker and local development.
- Configuration should be boring: stable runtime behavior in TOML, secrets and deployment wiring in env.
- Frontend flows should be dense enough for real work and calm enough for repeated use.
- Generated artifacts should be reproducible from commands, not manually curated.
